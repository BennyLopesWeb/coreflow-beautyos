# Release Notes — CoreFlow BeautyOS `2.12.0-r4-f9`

**Data:** 2026-07-22
**Codename:** R4-F9 Financeiro core-only + sunset pagamentos legado
**Branch:** `r4-f9-financeiro-pagamentos-sunset`

## Destaques

- **`confirmar_deposito_por_booking` popula `Financeiro`.** Na primeira
  confirmação de sinal de um booking core-only, o serviço registra
  entrada automática (`Sinal - Booking #{id}`) via
  `FinanceiroService.registrar_entrada_automatica` — paridade contábil
  com o path legado e base para sync de invoices (CF9).
- **Rotas `/pagamentos/sinal*`, `/pagamentos/comprovante*` → 410 Gone.**
  `LegacyGoneMiddleware` (R4-F1) passa a cobrir payments legado com
  successor `/v1/payments` (headers Deprecation/Sunset/Link).

## Sem breaking changes de API core

- `POST /admin/pagamentos/booking/{id}/confirmar-sinal` e `/v1/*` inalterados
  (exceto efeito colateral desejado: entrada Financeiro no deposit).

## Débito residual

- Reagendamento / pagamento final core-only completo.
- Clientes frontend que ainda chamam `/pagamentos/sinal*` recebem 410.
