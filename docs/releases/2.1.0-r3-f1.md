# Release Notes — CoreFlow BeautyOS `2.1.0-r3-f1`

**Data:** 2026-07-16  
**Codename:** R3-F1 Enforcement Expand  
**Branch:** `r3-f1-enforcement-expand`

## Destaques

- Enforcement **block** expandido: payments + fila legado → HTTP 409
- Production piloto: `APP_ENV=production` resolve para `block` (já previsto; agora com escopo ampliado)
- Booking legado permanece bloqueado (herança R2-F6)
- Invoices (`/financeiro/saida`) continuam em warn only

## Feature flags (defaults)

| Flag | Default |
|------|---------|
| `FEATURE_BOOKING_CORE_ENABLED` | false |
| `FEATURE_RESOURCE_ENGINE_ENABLED` | false |
| `FEATURE_PLUGIN_ENGINE_ENABLED` | false |
| `CORE_ENFORCEMENT_MODE` | off (local) / **block** (staging & production) |

## Breaking (staging / production)

Clientes que fazem `POST` em `/payments/*`, `/pagamentos/sinal` ou `/fila` recebem **409** com link para `/v1/payments` ou `/v1/waitlist`.

Ver [migration guide](../migrations/legacy-payments-fila-to-v1.md).

## Rollback

```bash
CORE_ENFORCEMENT_MODE=warn
```
