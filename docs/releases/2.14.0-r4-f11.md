# Release Notes — CoreFlow BeautyOS `2.14.0-r4-f11`

**Data:** 2026-07-22
**Codename:** R4-F11 Reagendamento core-only
**Branch:** `r4-f11-reschedule-core`

## Destaques

- **`POST /v1/bookings/{id}/reschedule`** — fecha booking `approved` como
  `rescheduled` e cria substituto no novo horário (mesmo customer/catalog/
  offering). Transfere `deposit_paid` / `payment_status`; se o sinal já
  estava pago, o novo booking nasce `approved`.
- Evento canônico **`booking.rescheduled`** (payload com `new_booking_id`).
- Frontend `reservationService.reagendar` aponta para a API v1.
