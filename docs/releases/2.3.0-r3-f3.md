# Release Notes — CoreFlow BeautyOS `2.3.0-r3-f3`

**Data:** 2026-07-20  
**Codename:** R3-F3 Remove Legacy Booking Write Routers  
**Branch:** `r3-f3-remove-legacy-write-routers`

## Destaques

- Removidos endpoints de **escrita** em `/agenda/agendamentos` e `/reservations` (RFC-003 M5)
- Leituras e `/agenda/disponibilidade` permanecem
- `QueueEntryService.aprovar_com_horario` cria booking via `CreateBookingHandler` (não mais `ReservationService`)
- Sucessor canônico: `/v1/bookings`

## Breaking

Clientes HTTP que ainda fazem `POST/PUT/DELETE` em `/agenda/agendamentos` ou `/reservations` recebem **405 Method Not Allowed**.  
Migrar para `/v1/bookings` (ver [legacy-booking-to-v1.md](../migrations/legacy-booking-to-v1.md)).

## Próximos passos (R4)

- `410 Gone` em rotas legado booking
- Remover `reservation.*` do catálogo de eventos
- Remover dual-write `project_*` / model `agendamentos`
