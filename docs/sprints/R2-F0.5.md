# Release 2 — Sprint R2-F0.5

**Versão:** `1.18.5-r2-f0.5`  
**Status:** ✅ Concluído  
**RFC:** [RFC-003](../rfc/RFC-003-CoreDomainConsolidation.md)  
**ADRs:** ADR-009, ADR-029, ADR-030

---

## Objetivo da fase

Wire ACL nos commands CQRS de booking — **Command → Port → Adapter → Legacy** — sem alterar comportamento e **sem** implementar Booking Domain puro (R2-F1).

---

## Escopo IN

| Entrega | Detalhe |
|---------|---------|
| Commands usam `LegacyBookingAdapter` only | create, approve, reject |
| Zero `ReservationService` em commands | FF-BKG-001 WARN |
| Zero `LegacySyncService` em commands | FF-BKG-002 WARN |
| `SchedulingPort` interface + ACL adapter | ADR-029 estágio 1 |
| Métodos ACL extendidos | approve, reject, sync booking |
| Testes regressão + forbidden imports | paridade comportamental |

---

## Escopo OUT (proibido nesta fase)

- Booking aggregate / domain service (R2-F1)
- Resource Engine (R2-F3)
- Plugin Engine (R2-F4)
- PaymentQueryPort (R2-F2)
- Idempotency (R2-F1b)
- Remover código legado
- Feature flags novas
- Alterar ADRs
