## Objetivo

Concluir a migração do domínio de booking (RFC-003 M4) removendo a delegação de escrita a `ReservationService` a partir dos commands CQRS e da ACL (`LegacyBookingAdapter`). O booking passa a ser **core-only**: não existe mais fallback para o path legado quando a flag está desligada — ela se torna um kill-switch de emergência.

## Mudanças de comportamento

### Commands de booking (`create/approve/reject/cancel`)

- **Antes (R2):** flag OFF → path ACL (`_execute_legacy_path`) delegava a `ReservationService`.
- **Agora (R3-F2):** flag OFF → `BusinessRuleError` imediato — nenhuma escrita ocorre. Roteadores `/v1/bookings/*` mapeiam `BusinessRuleError` para **HTTP 409** apontando para `/v1/bookings`.
- Flag ON (default) segue o path core (`_execute_core_path`) inalterado, exceto que **não publica mais** `reservation.*_alias` no outbox — apenas `booking.*`.

### ACL (`LegacyBookingAdapter`)

- `create_booking_via_legacy`, `approve_booking_via_legacy`, `reject_booking_via_legacy`, `cancel_booking_via_legacy` — sempre `_track()` + `logger.warning()` + `raise BusinessRuleError("Legacy booking write removed (R3-F2) — use /v1/bookings")`.
- `project_*` (dual-write outbound), `resolve_legacy_ids`, `sync_booking_from_agendamento`, `get_booking_legacy_id` — **inalterados**, continuam servindo o path core.

### `ReservationService`

- `criar`, `criar_de_schema`, `aprovar`, `rejeitar`, `cancelar` — corpo substituído por `raise BusinessRuleError(...)`. Assinaturas/DocStrings mantidas por compatibilidade de import.
- `listar`, `obter`, `confirmar_deposito`, `concluir`, `solicitar_reagendamento`, `aceitar_reagendamento`, `registrar_pagamento_final` — **mantidos e funcionais** (usados pelo ciclo de pagamento/dual-write).

### Eventos (ADR-027 sunset)

- Commands core não chamam mais `reservation_created_alias`, `reservation_approved_alias`, `reservation_rejected_alias`, `reservation_cancelled_alias`.
- As factories continuam existindo em `domain/events.py` (para o catálogo de eventos / eventual consumo em R4) mas emitem `logger.warning` se alguém as chamar.

## Rollback

```bash
export FEATURE_BOOKING_CORE_ENABLED=false
```

**Atenção:** diferente de releases anteriores, isso **não** restaura o path legado — passa a bloquear toda escrita de booking com `409 BusinessRuleError`. É um kill-switch de emergência (ex.: suspender temporariamente novas escritas), não um mecanismo de rollback funcional. Rollback de código real exige `git revert` do merge commit desta sprint.
