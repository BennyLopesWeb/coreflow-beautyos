## 2. Objetivo

Introduzir **`BookingDomainService.create()`** (e aggregate `Booking`) como **novo caminho de escrita** do domínio Booking quando `booking.core.enabled=true`, mantendo comportamento **idêntico** ao legado quando a flag está off.

Com flag **ON** (ADR-024):

- **Core** (`core_bookings`) = Source of Truth  
- **Legado** (`agendamentos`) = projeção outbound na **mesma transação** (ADR-025)  
- Evento **`booking.created`** publicado via outbox; alias **`reservation.created`** conforme ADR-027  

**Resultado mensurável:** P01, P02 e P09 PASS com flag ON e OFF; zero regressão vs R2-F0.5.

## 7. Fluxo aprovado

### Flag OFF (default — regressão idêntica F0.5)

```
HTTP POST /v1/bookings
    ↓
CreateBookingHandler
    ↓
LegacyBookingAdapter (ACL)
    ↓
Legacy Sync + Outbox booking.created (comportamento atual)
    ↓
Commit
```

### Flag ON (core path — esta sprint)

```
HTTP POST /v1/bookings
    ↓
CreateBookingHandler
    ↓
BookingDomainService.create()
    ↓
SchedulingPort.check_availability()
    ↓
Booking Aggregate (factory / create)
    ↓
BEGIN TRANSACTION
    ↓
CoreBookingRepository.save()          ← SoT
    ↓
LegacyBookingPort.project_create()    ← projeção agendamentos
    ↓
Outbox: booking.created
    ↓
Outbox: reservation.created (alias)
    ↓
COMMIT
    ↓
[Async] Outbox worker publish
```

**Falha na projeção legado → ROLLBACK total** (ADR-025).

## 14. Rollback

### Gatilhos

| Gatilho | Ação |
|---------|------|
| P01/P02/P09 FAIL | Flag OFF + investigar |
| Projection failure rate > 0 | Flag OFF all tenants |
| P1 produção | Flag OFF + revert PR |

### Procedimento completo

1. **Imediato (< 5 min):** `FEATURE_BOOKING_CORE_ENABLED=false` em todos os ambientes afetados  
2. **Validar:** POST `/v1/bookings` via ACL path — paridade F0.5  
3. **Se código defect:** `git revert <merge-commit-r2-f1>`  
4. **Schema:** Alembic downgrade apenas se migration F1 aplicada e revertida  
5. **Comunicar:** Registrar incidente em `ArchitectureDecisionLog.md` se P1  
6. **Versão:** Restaurar `APP_VERSION=1.18.5-r2-f0.5` se deploy revertido  

Orphans em `core_bookings` com flag OFF são aceitáveis (ADR-024) — reconciliation F5.
