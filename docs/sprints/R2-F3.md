## 2. Objetivo

Consolidar o motor de recursos reserváveis como **primeira classe** na plataforma, conforme [ResourceEngine.md](../ResourceEngine.md) e [ADR-007](../adr/ADR-007-ResourceEngine.md).

Com flag **`resource.engine.enabled`** ON (ADR-032):

- Módulo `modules/resource/` com domain / application / infrastructure  
- `ResourcePort` + `ResourceQueryPort` + `ResourceAllocationPort`  
- Scheduling consome `ResourcePort` — **zero `tranca_id`** em código novo fora ACL (FF-SCH-001)  
- API `POST/PUT/PATCH /v1/resources` (CRUD v1) além do GET existente  
- `resource_types` declarados no manifest beauty (e stubs sports/clinic)  
- Booking create com `resource_id` opcional valida conflito via engine  
- Eventos `resource.created`, `resource.updated`, `resource.allocated`, `resource.released`  

Com flag **OFF**: comportamento idêntico ao pré-F3 (SchedulingPort + ACL tranca); zero regressão P01–P09, P12, P06, P07.

**Resultado mensurável:** **P11 PASS** flag ON e OFF; FF-SCH-001 ERROR; FF-RES-001 WARN; pytest ≥305 + novos F3; `APP_VERSION=1.21.0-r2-f3`.

## 8. Ordem de implementação (obrigatória)

```
(0) Flag FEATURE_RESOURCE_ENGINE_ENABLED + feature_flags registry
        │

        │
        v
(4) Router /v1/resources CRUD — wire ao módulo resource
        │
        v
(5) ResourcePort adapter — implementação estágio 2 ADR-029
        │
        v
(6) SchedulingAvailabilityService branch flag ON/OFF
        │
        v
(7) Booking create — resource_id + conflict check (P11)
        │
        v
(8) Eventos resource.* + outbox
        │
        v
(9) Plugin manifests resource_types (beauty + stubs)
        │
        v
(10) Paridade P11 + regressão suite
```

### Diagrama — create booking com resource (flag ON)

```
POST /v1/bookings { resource_id?, scheduled_at, ... }
        │
        v
Resolve offering duration + ends_at
        │
        v
SchedulingPort / ResourcePort.check_availability(resource_id, slot)
        ├─ conflito → 409 (P11)
        └─ ok → continuar F1 create path
        │
        v
Persist booking + allocate resource (se aplicável)
        │
        v
Outbox: booking.created + resource.allocated (se alocado)
```
